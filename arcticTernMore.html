The following time formats are supported:
  • ANSI SQL Time
  • Google Chrome (WebKit) Time
  • Java Time
  • Julian Day
  • Apple HFS+ Time
  • Apple Absolute (Cocoa) Time
  • Microsoft Excel Time
  • Microsoft DOS / FAT File Time
  • SQLite3 Time (real and integer)
  • Unix Time (32/64 bit, signed/unsigned, Decimal 10 and Decimal 13)
  • Microsoft Windows FILETIME
  • Microsoft Windows OLE Time

Time Zones and Daylight Savings Time Bias are also supported.

As an example, the value 591546428 may represent an Apple Absolute (NSDate) Time for 2019-09-30 14:27:08 (GMT).  The same time is represented by the Windows FILETIME value 132143272280000000 and MS-DOS (FAT) value 1935953726.

By default, time values are represented in ISO 8601 format and are displayed with a precision of 100 nanoseconds.  The local time zone is retrieved from the Operating System by default, and Daylight Savings Times are not set.  These metrics can be modified via the Settings menu.

The User Interface consists of three sections, a list of time conversions, difference calculator and time arithmetic.

The Tool Bar provides shortcut controls for the conversion list.

Difference Calculator

Calculate the elapsed time between two time values (always returns a positive value).

Example 1 - Duration of 2004
2004-01-01 00:00:00 <=> 2005-01-01 00:00:00
Result:
    31622400 seconds
    366 days
    1 year
Example 2 - Duration between 2.28.05pm and midnight on the 1st of October 2019
2019-10-01 14:28:05 <=> 2019-10-01 00:00:00
Result:
    52085 seconds
    0.6028 days
    14 hours, 28 minutes and 5 seconds

Time Arithmetic

Performing arithmetic on time is achieved through a calculator with a number of simple binary and unary operators:

  • '*' - multiplication
  • '/' - division
  • '+' - addition
  • '-' - subtraction
  • div - integer division
  • mod - (C) modulus (result always has the same sign as the dividend)
  • modxl - Microsoft Excel modulus (returns a result with the same sign as the divisor)
  • [seconds, second or sec] - X seconds
  • [minutes, minute or min] - X minutes
  • [hours, hour or hrs] - X hours
  • [days or day] - X days
  • [weeks, week or wks] - X weeks
  • [months or month] - X (calender) months
  • [years, year or yrs] -X (calendar) years
  • dow -Day of Week

The TIME class (reserved word 'TIME') can be assigned a time value (containing four properties: day, month, year, dow).

Example 1 - Calculate the date for Easter Sunday 2016
Let TIME = 2016-4-1
(((((TIME / 86400 - 15) / 7 + ((19 * (TIME.year modxl 19) - 7) modxl 30) * 0.14) + 0.5) div 1) * 7 + 9) days
Result: 2016-03-27 00:00:00 (27th of March 2016)
Example 2 - Calculate Daylight Savings Time (AEDT) 2019/2020
Starting (2019):
Let TIME = 2019-1-1
TIME + ((7 - TIME.dow) mod 7) day + 39 weeks + 2 hours + ((((TIME + ((7 - TIME.dow) mod 7) day + 39 weeks).days) + 1) div 30) weeks
Result: 2019-10-06 02:00:00 (2am 6th of October 2019)
Ending (2020):
Let TIME = 2020-1-1
TIME + ((7 - TIME.dow) mod 7) day + 13 weeks + 3 hours - ((((TIME + ((7 - TIME.dow) mod 7) days + 13 weeks).days) - 1) div 7) weeks
Result: 2020-04-05 03:00:00 (3am 5th of April 2020)

Time Conversion

Each column of the 'Conversion List' has a context-menu (right-click):

Encoded Value

Decimal/Hexadecimal - values can be entered and/or displayed using either decimal of hexadecimal number system.  If a value is entered with the prefix '0x' then the number system is presumed to be hexadecimal.  Endianness can only be set for a value using the hexadecimal number system.

Endian

The value is interpreted as either Big or Little Endian.  If the field is blank, the value is interpreted as big endian.

Time Zone

The encoded value is interpreted as having a time bias based on the Time Zone value (relative to Greenwich Mean Time).  Generally, this value is set to GMT.

Format

Encoding format.

GMT and Local Time

Encoded value expressed as a time (GMT and local time).  The display format and precision can be changed via the Settings menu.

Explanation

Each row of the 'conversion list' is an equivalence equation () with the left, 'Encoded value', 'Endian' and 'Time Zone' equaling the time displayed on the right, using the 'Format' encoding (GMT and local).  When an element on one side is changed, an element on the other side is changed to balance the equation. 

For example, the value 567993600 encoded using the Apple Absolute Time equates to GMT 2019-09-30 14:27:08:

When the 'Format' is changed to 'Unix Time' (signed 32-bit), the 'Encoded Value' is re-calculated to maintain equivilence.

That is, the value 591546428 encoded as 'Apple Absolute Time' is equivalent to 1569853628 encoded as 'Unix Time'.

When entering a time, the shortcut 'n' is interpreted as Now (current time), 't' is Today, and 'y' is Yesterday.

UTC vs. GMT

Times relative to Greenwich Mean Time (GMT) are used to display local times, not Coordinated Universal Time (UTC).  Since UTC is a standard, there is no time zone, territory or country that uses it for a local time.  GMT is an actual time zone.
