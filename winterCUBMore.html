winterCUB using the Windows VirtDisk API to mount each pseudo file as a physical drive.

winterCUB utilises the software interface Filesystem in User-space (FUSE) for Unix-like Operating Systems. In order to use winterCUB, Windows File System Proxy (WinFsp) must be installed.

winterCUB requires Administrator privileges and launches as a persistent application (controlled via the System Tray).

To install WinFsp, execute the winfsp-x.x-xxx.msi installer and follow the prompts. Only the Core feature must be selected (the Developer and FUSE for Cygwin options are not necessary). The installer can be downloaded from here.

Control Panel

A Control Panel is accessible via the context-menu of the winterCUB icon found on the System Tray.

The panel displays information about the attached computer and a list of attached devices. If the host is nanoq, mountable volumes are listed for each attached device (right-click to mount/unmount).

Three action buttons are provided:

Open Folder

Opens the folder containing the pseudo folder presenting the attached block devices for the particular WET cable.

Connect/Disconnect

Connect/disconnect to/from the attached computer the the particular WET cable.

Open/Close Console

Open/close a console which represents the arcticBear host application as seen on the host computer. The console provides a user interface to the host application (not applicable when nanoq is the host).
