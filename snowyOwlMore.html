Since the Windows Operating System can boot from any NTFS formatted partition, snowyOwl must first identify that partition.  The application command-line will accept one or more physical drives (‘\\.\PhysicalDrive#’), physical volumes (‘\\.\X:’), raw images of a drive or partition, or Expert Witness Disk Image / Encase image file format (E01) file containing a drive or partition.

Regardless of whether the Windows OS boots via UEFI or legacy BIOS, snowyOwl attempts to detect the OS system folder.  If this cannot be determined and the system option is not set, the root folder 'Windows' is assumed.

The Windows Registry is then parsed to identify the values of variable paths using the KNOWNFOLDERID constants (not including independent vendor defined folders).  This allows snowyOwl to conduct an exhaustive, heuristic search for Application User Model Identifiers for the particular OS installation.

The search for applications encompasses all partitions provided to snowyOwl that are formatted in the supported file systems.

Finally, both the automatic and custom destinations for each installed User are parsed and collated to provide a detailed Jump List.  This includes, where possible, a detailed list of the applications referenced by the Jump List entries.  Windows Unified Platform (UWP) applications are supported.


The results can be reported in the form of an HTML document.  Jump List entries are reported in the same order as the tables in the User Interface (column headers determine the sort order of each list).  Target files can be included in the report, where applicable, and are accessible via hyperlinks in the report tables.  Be aware, this may require significant storage.

The report header can be modified via snowyOwl.ini.


Report Table Columns

Identifier (AppID)

The calculated application identifier.

Name

Application name.

Volume Id

The volume identifier on which the target file/folder was last found.

Access Count

Number of times that the target has been accessed.

Pinned

Pin status of the target.

Last Accessed

The (local) time that the target was last accessed.

Target Created

The (local) time that the target file/folder was created.

Target Modified

The (local) time that the target file/folder was last modified.

Target Accessed

The (local) time that the target file/folder was last accessed.

Target Size

Size of the target file/folder (bytes).

Target File Path

Path of the target file/folder.

Shortcut Command Line

The command-line argument(s) used to access the target.

Type

The destination type (automatic or custom).



snowyOwl is a stand-alone application that does not require installation. It does however, require the Windows File System Proxy (WinFsp).
