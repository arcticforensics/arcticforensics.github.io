The arcticBear desktop is partitioned into four panels. The two primary partitions list Source and Destination devices, a status panel displaying the write-block state and battery level (if applicable), and a forth panel with content depending on which item in the two lists is actively selected.

Source

The Source lists attached block devices and pseduo block devices such as block device images (including raw and Encase/EWF files), and RAIDs. Device labels are constructed in the form [type][index]<partition>.

sdLinux block device
ldmMicrosoft Logical Disk Management (dynamic disks)
raidSoftware RAID (e.g. isw, Intel software RAID)
vdVirtual block device (drive image)
srOptical drives

Write-blocking: arcticBear relies on the arcticLinux kernel to provide block device write-blocking. All attached devices are write-blocked at boot time, and cannot be written to unless detached. An unattached block device can be made writable by the user and will appear in the Destination list.

RAID/LDM: Software RAID sets and Microsoft's dynamic disks are identifed and parsed accordingly. Once a complete set is identifed, and pseudo 'raid' block device is emulated representing the RAID/LDM set.

WET: Supported transfer cables (USB Host-to-Host) are displyed in the Destinations list when attached and provide a Host/Client paradign with Microsoft Windows based systems running the winterCUB client software (et al.). All Source block devices are made available to the client system.

Desintation

Sink devices such as mountable file systems (Ext, NTFS, FAT, HFS, APFS, etc.) on block devices, network (Ethernet, Wifi), iSCSI and Windows Easy Transfer (WET) cables are listed as Destination devices.
