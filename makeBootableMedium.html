The ISO images can be 'burnt' to an optical disc using any of a myriad of authoring tools.  The raw image files require a software tool capable of writing directly to a block device. The dd command can be used.  For example:

Unix-like OS
dd if=arcticLinux_Intel_x86_EFI_1.9.bin of=/dev/sdc bs=1M

Microsoft Windows
dd if=arcticLinux_Intel_x86_EFI_1.9.bin of=\\.\PhysicalDrive3 bs=1M

Since the EFI and Hybrid bootable images are based on the GPT schema, it is necessary to make sure a backup of the Partition Table exists at the end of the target medium. The sgdisk command can be used. For example:

Unix-like OS
sgdisk -e /dev/sdc

Microsoft Windows
sgdisk -e \\.\PhysicalDrive3

The Unix-like commands dd and sgdisk are included in Console Tools as Windows ports. Note that the Microsoft Windows Operating System restricts direct disc writes to block devices containing one or more file systems.  As such, the target medium should be 'erased' before writing the image.  See How to Diskpart Erase/Clean a Drive Through the Command Prompt or PowerShell Clear-Disk.
