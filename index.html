Welcome to Arctic Forensics.

arcticBear, the primary product from Arctic Forensics, is an all-in-one solution for creating forensic copies of digital storage media, both attached and removable devices  The application shares a symbiotic relationship with arcticLinux, a forensic Operating System based on the Linux kernel.

A number of other products are also available, each created to fulfill a specific task.

All products are provided as-is, open-source, and are subject to the GNU General Public License. All Source code is made available directly, where available, otherwise links to sources are provided.  There is no proprietary software involved with any Arctic Forensics product.

arcticBear and nanoq (Release 1.13) are available as boot disc image files, based on a 512-byte sector size, and optical media (ISO).

Two common methods of booting a computer are via the Basic In/Output System (BIOS) or the (Unified) Extensible Firmware Interface (UEFI).  BIOS, also known as Legacy, has been ostensibly superseded by (U)EFI.  Although not universal, BIOS systems traditionally boot from drives formatted with the Master Boot Record (MBR) schema, while (U)EFI systems generally boot from drives with the GUID Partition Table (GPT) schema.

Some modern systems support both Legacy and (U)EFI paradigms.  Support for Legacy systems by (U)EFI is often referred to as the Compatibility Support Module (CSM).

In general, a hybrid boot image will successfully boot many modern systems.  The hybrid schema includes a GUID Partition Table and a Master Boot Record.  This may cause confusion with some firmware, hence the reason why arcticLinux is provided in a combination of schemas and architectures.

Please note that only systems with the Intel x86 processing unit are supported at this time.  All Intel x86 images include both 32 and 64 bit architectures.  The El Torito images are for bootable optical media (ISO 9660 CD-ROM).


Forensic Tools includes arcticExplorer, arcticTern, snowyOwl, arcticLynx, caribou, snowGoose, narwhal, ermine, arcticFox and Console Tools. The Microsoft Windows binaries (version 20160918) and (Intel) Macintosh package (version 20171104) for Joachim Metz's libewf is also available.

FAQ


How do I create an arcticLinux bootable medium?

The arcticLinux OS (containing arcticBear/nanoq) is available as an ISO (for optical media) or raw disk image (for boot discs). The commands dd (and sgdisk) on UNIX-like OS can be used or similar commands found in the Console Tools (Windows ports).

FAQ


Booting a Secure Boot System

arcticLinux supports (U)EFI Secure Boot (64-bit systems only). Although not digitally signed, the arcticLinux kernel is authenticated via an (U)EFI pre-loader provided through the Linux Foundation Secure Boot System (see James Bottomley's blog).

The pre-loader is used to enroll the hash of the "loader" EFI-stub, which contains arcticLinux.  Enrolling the appropriate hash requires user interaction.

If the arcticLinux EFI-stub is not enrolled on the particular firmware, the (U)EFI system will launch the 'preloader' which provides a basic user-interface. The user must then enroll the hash by selecting the 'loader.efi'.

arcticBear

Although arcticBear is an application, the product is packaged with the arcticLinux forensic Operating System which includes all of the necessary supporting tools. arcticLinux is based on a minimalist Linux kernel which excludes a Graphical User Interface (GUI). The arcticBear User Interface is text based and driven using a standard AT 84-key keyboard.

The package is supplied as a bootable medium for Intel x86 based computers (32 and 64 bit) and is capable of booting BIOS (legacy) and (U)EFI systems.

nanoq

A Target Mode emulator based upon arcticBear.

nanoq is an alternative to arcticBear with reduced functionality that requires no user interaction. As with arcticBear, nanoq is launched after the arcticLinux OS boots. The Linux frame buffer device console is used as a pseudo graphical interface. For those systems that do not support frame buffer, a series of instructions are displayed on the text console.

The graphical interface displays the USB symbol () for each supported USB Host-to-Host transfer cable connected (at least one symbol is displayed to indicate that the application is ready). A particular USB symbol is displayed as either grey, yellow or green. A single grey symbol indicates that no supported cables are connected. A yellow symbol represents a supported, connected cable, and a green symbol indicates that a successful handshake with a relevant client machine has been achieved and data transfer is active.

winterBear

The Microsoft Windows equivalent to arcticBear is winterBear. When launched, the Windows Management Instrument (WMI) is used to enumerate all drives/volumes. This includes collating Dynamic drives (using the Logical Disk Manager ldmtool) and mounted pseudo volumes (such as TrueCrypt/VeraCrypt and BitLocker volumes).

The application auto-detects when a USB Windows Transfer Cable (WET) is connected and attempts to hand-shake with an instance of the client application (winterCUB). Only the Prolific chip-sets PL25A1 and PL27A1 are supported for cable transfer but no device driver software is required. As a consequence, systems with a Prolific WET cable driver installed (such as Cellebrite's Physical Analyzer) will obscure cable access for both winterBear and winterCUB. See "How to remove Cellebrite transfer cable driver".

winterCUB

The client application for Microsoft Windows, winterCUB (Windows Client via USB Bridge), is the client side of arcticBear and winterBear. winterCUB functions exclusively with a Windows Easy Transfer cable (USB Host-to-Host cable) to communicate with the host computer. All block devices and logical volumes from the host computer are exposed as read-only pseudo files accessible through the winterCUB icon on the Desktop.

The winterCUB Desktop icon represents a pseudo folder containing folders named after the Plug and Play Identifier of each compatible transfer cable connected. That is, a single instance of winterCUB can manage multiple hosts. Deleting the Desktop icon will terminate the application.

A software driver for transfer cables is not used by Microsoft Windows 8 and above. The WinUSB API is utilised instead. In the case of cables containing Prolific PL25A1 and PL27A1 chip-sets, some products such as Cellebrite's Physical Analyzer install drivers that interfere with winterCUB. See "How to remove Cellebrite transfer cable driver".

arcticExplorer

A Microsoft Windows port of FUSE based file system projects.

The Graphical User Interface (GUI) arcticExplorer provides a simple interface to mount and access (read-only) the file systems: Ext 2/3/4, HFS/HFS+, APFS, NTFS, FAT 12/16/32 and ExFAT.  Theses file systems are accessible from physical mass storage media or file containers such as Apple's DMG, Expert Witness Format / Encase Image File Format (EWF, E01) and raw images. This includes encrypted volumes using BitLocker, LUKS, TrueCrypt/VeraCrypt and APFS encryption.

arcticExplorer can also mount Zip archives, Windows Registry Hives and File Transfer Protocol (FTP) connections as pseudo file systems.

All utilise the software interface Filesystem in User-space (FUSE) for Unix-like Operating Systems. In order to use these projects on the Microsoft Windows Operating System, Windows File System Proxy (WinFsp) must be installed.

arcticExplorer requires Administrator privileges and launches as a persistent application (controlled via the System Tray).  If the 'Auto Mount' option is set, arcticExplorer will mount supported file systems found on the arrival of new block devices.

Although file systems NTFS and Ex/FAT are supported natively by Windows, arcticExplorer does not rely on the Operating System so file attributes such as security permissions, system and hidden bits are ignored.  Each file system is mounted as read-only making arcticExplorer forensically sound.

arcticTern

A time storage conversion utility.

Time values can be stored using numerous formats and vary by metrics such as epoch and unit of measure, One of the more common formats originated from the Unix Operating System (OS), storing a time value as the number of seconds since midnight on the 1st of January 1970. Many are stored as integers while others use floating point. Additionally, endianness varies between Operating Systems.

arcticTern was created as a utility to convert between formats and identify which format a time value may be using. The application also includes a time difference calculator and a time arithmetic calculator.

snowyOwl

A Microsoft Windows Jump List (Destination) parsing and reporting application.

Usage

snowyOwl [-s|--system <system_folder>] <source_1 [-o|--offset <offset>]> [[source_2] [-o|--offset <offset>]] ...

The application accepts one or more drives and/or volumes. In each instance, snowyOwl will attempt to determine the partition containing the Windows Operating System. Only partitions or volumes containing the NTFS, FAT16/32 or ExFAT file systems are examined.

The offset option is ordinarily not required, but can be used to restrict snowyOwl to parsing a single partition within a source, for example. Likewise, the system option is used to identify the Microsoft Windows system folder only when snowyOwl is unable to identify it. Each parameter can be a physical entity, an Encase Image File Format (E01) or a raw image file.

snowyOwl can be launched, for example, by dragging one or more E01 files onto the application. If the application is launched without any parameters, the user is prompted to load a saved Profile.

arcticLynx

A HEIF media file viewer/converter.

The High Efficiency Image Format (H.265) is the successor to the Advanced Video Coding (H.264) media format defined as Part 12 within the MPEG-H media suite (ISO/IEC 23008-12).

arcticLynx was created as a utility to view and convert High Efficiency Video Coding (HEVC) and High Efficiency Image Coding (HEIC) media files.  Specifically, audio/video container files encoded using HEVC can be converted to Audio Video Interleave / MPEG-2 media files, while HEIC image files can be converted to JPEG, WebP, PNG or BMP (and vice versa).

The audio/video conversion is not optimised as it simply decodes then recodes each frame of the default streams.

As may be the case, arcticLynx loads all of the image streams for those HEIC files containing either a single image, a collection of images, or sequences of images.

The application uses a subset of the FFmpeg libraries for audio/video and libheif for still image files.

arcticLynx is a stand-alone application that does not require installation.

caribou

A media file metadata and codec parser.

caribou displays codec information (format, dictionary, streams, etc.) and metadata (TIFF/Exif/GPS tags, XMP, PrintIM, ICC Profile and IPTC IIM) as a tree-view to describe the media metrics.

Information is grouped by source:


file system

Basic information (file name, path, size and time stamps) retrieved from the file system hosting the file.

metadata

Tag information returned from ExifTool (libheic is used for HEIC files, see streams below).

format

Information retrieved via FFmpeg's AVFormat library.

dictionary

Tag information retrieved via FFmpeg's AVFormat library.

streams

Stream information retrieved via FFmpeg's AVFormat library. In the case of HEIC files, libheic is used to retrieve stream data including the metadata for each stream.



The application uses the FFmpeg libraries to parse codec information and Phil Harvey's ExifTool to extract internal metadata (read-only). Exif tags are described via Aware System's TIFF Tag Search. caribou is a stand-alone application that does not require installation. Target files can be parsed by dragging and dropping it onto the executable.

snowGoose

A media file recovery application for Hikvision and RaySharp (carried by Swann et al.) Digital/Network Video Recorders (domestic and small business closed-circuit television security systems).

Both systems record audio/video media files using their repsective proprietary disk and file system formats. snowGoose parses the stored data from either physical devices (such as those removed from the DVR units) or copies of same (EWF or raw). The resulting files are presented in a pseudo folder on the user's Desktop. To date, both systems store video as either H.264 (AVC) or H.265 (HEVC) raw stream data. Hikvision provides VSPlayer for audio/video playback of their raw data, similar to RaySharp's video player. Alternatively, ffplay from the ffmpeg library suit or VLC can be used.

A simple selection dialog is provided when the application is executed, or a copy of the source device can be dropped onto the application.

ermine

A parser for the Microsoft Windows application YourPhone.

ermine [-s|--system <system_folder>] [-a|--appdata <application_data_folder>] <source_1 [-o|--offset <offset>]> [[source_2] [-o|--offset <offset>]] ...

The application accepts one or more drives and/or volumes. In each instance, ermine will attempt to determine the partition containing the Windows Operating System and the folder containing the YourPhone application. Only partitions or volumes containing the NTFS, FAT16/32 or ExFAT file systems are examined.

The offset option is ordinarily not required, but can be used to restrict ermine to parsing a single partition within a source, for example. Likewise, the system option is used to identify the Microsoft Windows system folder only when ermine is unable to identify it. In the event that only the application data folder is avaialble, the appdata option can be used (overiding the system and offset options). Each parameter can be a physical entity, an Encase Image File Format (E01) or a raw image file.

ermine can be launched, for example, by dragging one or more E01 files onto the application.

The application does not currently support mobile device applications.

arcticFox

A Microsoft Windows software write-blocker for attached drives (USB, SATA, virtual, etc.)

arcticFox [-s|--onstart protect=<on|off>] [-c|--onclose protect=<on|off>]

arcticFox can be launched from the command line to set the type of protection when the application starts and when it closes.

Except for USB devices, write protection is not persistant while the application is not active. A complete Help file accompanies the application.

arcticWolf

An Apple APFS file system decryptor.

arcticWolf <source>

arcticWolf can decrypt Apple's APFS file system using either password or recovery key. The source can be a physical drive, raw drive image or Expert Witness Format / Encase Image File Format (EWF, E01). Hardware encrypted volumes (T2 and M1) are not supported.

Once processed, the original APFS source is exposed as a virtual volume with decrypted data, on-the-fly. A profile file is produced which can later be used along with the original source to reproduce on-the-fly decrypted data.

Console Tools

The following command-line tools were created for Microsoft Windows;
raw2VHD*
Creates a pseudo file representing a Microsoft Virtual Hard Drive (VHD) of a source file / device. By default, the VHD pseudo file appears in a unique mount point on the Desktop. The VHD is 'unmounted' when the mount point is deleted. No changes are made to the source file / device.

Usage: raw2VHD <raw file / device> [mount point]
ro2w*
Creates a pseudo, writable file copy of a read-only file/device. All writes to the pseudo file are written to a temporary file (in the default temporary folder). No changes are made to the source file/device. By default, the writable pseudo file appears in a unique mount point on the Desktop. The writable file is 'unmounted' and temporary file deleted when the mount point is deleted.

Usage: ro2w <source file/device> [mount point]

* Requires Windows File System Proxy.

Understanding Windows Jump List

A forensic operating system based on the Linux kernel, arcticLinux provides kernel-level write-blocking for block devices.

Standard USB use a master/slave paradigm; a host acts as the master device, and a USB device acts as a slave. For a standard USB implementation, devices must assume one role or the other, with computers generally set up as hosts. A USB Host-to-Host Bridge allows a computer to perform both master and slave roles, similar to USB On-the-Go (OTG), providing a direct peer-to-peer data transfer connection between two computers via USB.

NTLM authenticates clients and servers via a challenge-response method composed of three messages. NTLM uses one or more server-stored hashed passwords for authentication. Each password value is a 16-byte LM hash or NT hash.
